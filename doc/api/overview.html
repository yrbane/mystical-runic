Référence de l'API de Mystical-Runic

Cette section fournit une documentation détaillée des traits, structures, énumérations et fonctions publiques exposées par la bibliothèque Mystical-Runic. Elle est conçue pour être une référence complète pour les développeurs souhaitant intégrer et étendre les capacités de templating dans leurs applications Rust.

Principaux Composants de l'API

Les composants suivants constituent le cœur de l'API de Mystical-Runic, vous permettant d'interagir avec le moteur de template, de gérer les données et de gérer les erreurs :

  • TemplateEngine (alias RuneEngine) : Le moteur principal pour charger, compiler et rendre les templates. C'est votre point d'entrée pour toutes les opérations de templating.
  • TemplateContext (alias RuneScroll) : Le conteneur de données passé aux templates. Il organise les données sous une forme hiérarchique accessible par le moteur de rendu.
  • TemplateValue (alias RuneSymbol) : Représente les différents types de valeurs de données (chaînes, nombres, booléens, tableaux, objets) dans le contexte du template. C'est l'abstraction des données pour le moteur.
  • TemplateError (alias RuneError) : Le type d'erreur standard retourné par les opérations de templating. Il fournit des informations détaillées sur les problèmes rencontrés.
  • FilterFunction (alias MysticFilter) : Un type de fonction pour définir des filtres personnalisés, permettant de transformer les données directement dans les templates.
  • HelperFunction (alias AncientHelper) : Un type de fonction pour définir des helpers personnalisés, offrant des capacités d'extension plus complexes pour les templates.

Autres Exportations Publiques

La bibliothèque exporte également d'autres types et fonctions pour des cas d'utilisation avancés, qui sont souvent activés via des fonctionnalités optionnelles de Cargo :

  • DebugInfo, DebugRenderResult, ExecutionStep, PerformanceMetrics (alias RuneTrace, RuneDivination, RuneStep, RuneMetrics) : Types liés au débogage et à l'analyse des performances du rendu des templates.
  • LspParseResult, TemplateBlock, CompletionItem, SyntaxToken, Diagnostic, HoverInfo, DefinitionInfo (alias RunicLore, RunicBlock, etc.) : Types pour l'intégration avec les serveurs de langage (LSP) pour une meilleure expérience de développement dans les IDE.
  • AsyncTemplateEngine (alias AsyncRuneEngine) : Le moteur de template asynchrone, disponible si la fonctionnalité async est activée.
  • Intégrations spécifiques aux frameworks web (par exemple, AxumTemplateEngine, WarpTemplateEngine, ActixTemplateEngine) : Disponibles si la fonctionnalité web-frameworks est activée, elles fournissent des adaptateurs pour une intégration facile avec ces frameworks.
  • Types liés à WASM (par exemple, WasmTemplateEngine, WasmRuneEngine) : Pour la compilation et l'utilisation de Mystical-Runic dans des environnements WebAssembly, si la fonctionnalité wasm est activée.
