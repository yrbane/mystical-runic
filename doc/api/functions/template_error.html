Enum TemplateError (alias RuneError)

TemplateError est le type d'erreur principal utilisé par la bibliothèque Mystical-Runic pour signaler les problèmes survenus pendant les opérations de templating. Il encapsule diverses conditions d'erreur qui peuvent survenir lors du parsing, de la compilation ou du rendu des templates, offrant une gestion d'erreur robuste et informative.

Définition

Bien que les détails internes de l'énumération TemplateError puissent être complexes et inclure diverses variantes pour différents types d'erreurs (par exemple, erreurs de syntaxe, erreurs d'E/S lors de la lecture de fichiers, erreurs de données manquantes ou de type incorrect, erreurs d'exécution de filtres/helpers), elle est généralement exposée comme un type d'erreur qui implémente les traits standards de Rust comme std::error::Error et std::fmt::Display. Cela permet une gestion d'erreur cohérente et la possibilité d'afficher des messages d'erreur conviviaux.


pub enum TemplateError {
    // Erreurs de parsing (par exemple, syntaxe de template invalide)
    ParseError(String),
    // Erreurs d'E/S (par exemple, fichier de template introuvable)
    IoError(String),
    // Erreurs de données (par exemple, variable non trouvée, type incorrect)
    DataError(String),
    // Erreurs d'exécution (par exemple, problème dans un filtre ou helper personnalisé)
    RuntimeError(String),
    // Autres erreurs génériques
    Other(String),
}

// Implémentations pour Display, Error, etc. (simplifié)
impl std::fmt::Display for TemplateError {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        // ... logique d'affichage ...
        write!(f, "Erreur de template: {{:?}}", self)
    }
}

impl std::error::Error for TemplateError {}

Gestion des Erreurs

Les fonctions de rendu et de manipulation des templates dans Mystical-Runic retournent généralement un TemplateResult<T> (alias RuneResult<T>), qui est un alias pour Result<T, TemplateError>. Cela vous permet de gérer les erreurs de manière idiomatique en Rust en utilisant le pattern Result et l'opérateur ?.

Exemple de Gestion d'Erreur


use mystical_runic::{TemplateEngine, TemplateContext, TemplateValue, TemplateError};

fn main() -> Result<(), TemplateError> {
    let mut engine = TemplateEngine::new("templates");
    let mut context = TemplateContext::new();

    // Exemple 1: Erreur de syntaxe dans le template
    let invalid_template = "Hello {{ name }! "; // Manque un '}'
    match engine.render_string(invalid_template, &context) {
        Ok(rendered) => println!("Rendu : {{}}", rendered),
        Err(e) => eprintln!("Erreur de rendu (syntaxe) : {{}}", e),
    }

    // Exemple 2: Fichier de template manquant
    match engine.render_file("non_existent.html", &context) {
        Ok(rendered) => println!("Rendu : {{}}", rendered),
        Err(e) => eprintln!("Erreur de lecture de fichier : {{}}", e),
    }

    // Exemple 3: Variable manquante ou de type incorrect (peut être une DataError ou RuntimeError selon l'implémentation)
    let template_with_missing_var = "Hello {{ missing_var }}!";
    match engine.render_string(template_with_missing_var, &context) {
        Ok(rendered) => println!("Rendu : {{}}", rendered),
        Err(e) => eprintln!("Erreur de données : {{}}", e),
    }

    Ok(())
}
