Struct TemplateEngine (alias RuneEngine)

TemplateEngine est le cœur de la bibliothèque Mystical-Runic. C'est l'instance principale que vous utiliserez pour charger, compiler et rendre vos templates. Elle gère le cycle de vie complet du templating, de la lecture des fichiers à la génération du HTML final.

Création d'une Instance

TemplateEngine::new(template_dir: &str)

Crée une nouvelle instance de TemplateEngine. Le répertoire spécifié sera utilisé comme base pour résoudre les chemins relatifs des templates inclus ou étendus.

  • Paramètres :
    • template_dir : Le chemin du répertoire où se trouvent vos fichiers de template. Ce chemin est crucial pour que le moteur puisse trouver les templates référencés par {{include}} ou {{extends}}.
  • Exemple : 
    
use mystical_runic::TemplateEngine;

// Initialise le moteur avec le répertoire 'templates/' comme base
let mut engine = TemplateEngine::new("templates");

Rendu des Templates

TemplateEngine offre deux méthodes principales pour rendre les templates : à partir d'une chaîne de caractères ou à partir d'un fichier.

render_string(template_content: &str, context: &TemplateContext) -> TemplateResult<String>

Rend un template fourni directement sous forme de chaîne de caractères. Utile pour les templates courts ou générés dynamiquement.

  • Paramètres :
    • template_content : La chaîne de caractères contenant le template à rendre.
    • context : Une référence à un TemplateContext contenant les données à injecter dans le template.
  • Retourne : Un TemplateResult<String>. En cas de succès, contient le HTML rendu. En cas d'erreur (syntaxe invalide, données manquantes, etc.), retourne un TemplateError.
  • Exemple : 
    
use mystical_runic::{TemplateEngine, TemplateContext, TemplateValue};

let mut engine = TemplateEngine::new("templates");
let mut context = TemplateContext::new();
context.set("name", TemplateValue::String("World".to_string()));

let rendered = engine.render_string("Hello {{name}}!", &context).unwrap();
println!("{{}}", rendered); // Affiche: Hello World!

render_file(file_path: &str, context: &TemplateContext) -> TemplateResult<String>

Charge un template à partir d'un fichier spécifié par son chemin relatif (par rapport au template_dir de l'engine) et le rend. Idéal pour les templates stockés sur le système de fichiers.

  • Paramètres :
    • file_path : Le chemin relatif du fichier de template (par exemple, "greeting.html" si le fichier est dans le template_dir).
    • context : Une référence à un TemplateContext contenant les données.
  • Retourne : Un TemplateResult<String>.
  • Exemple : 
    
// Supposons que vous avez un fichier 'templates/greeting.html' avec 'Hello {{name}}!'
use mystical_runic::{TemplateEngine, TemplateContext, TemplateValue};

// Assurez-vous que le répertoire 'templates' existe et contient 'greeting.html'
// (Ces lignes sont pour l'exemple, vous les auriez normalement déjà)
// std::fs::create_dir_all("templates").unwrap();
// std::fs::write("templates/greeting.html", "Hello {{name}}!").unwrap();

let mut engine = TemplateEngine::new("templates");
let mut context = TemplateContext::new();
context.set("name", TemplateValue::String("Rustacean".to_string()));

let rendered = engine.render_file("greeting.html", &context).unwrap();
println!("{{}}", rendered); // Affiche: Hello Rustacean!

Enregistrement des Filtres et Helpers Personnalisés

TemplateEngine vous permet d'étendre ses capacités en enregistrant des fonctions de filtre et d'aide personnalisées. Ces fonctions peuvent être utilisées directement dans vos templates.

register_filter(name: &str, filter_fn: FilterFunction)

Enregistre une fonction de filtre personnalisée sous un nom donné. Les filtres transforment la valeur d'une variable.

register_helper(name: &str, helper_fn: HelperFunction)

Enregistre une fonction d'aide personnalisée sous un nom donné. Les helpers peuvent exécuter une logique plus complexe et interagir avec le contexte ou le moteur.
