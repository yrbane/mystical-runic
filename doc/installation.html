Installation de Mystical-Runic v0.5.2

Pour intégrer Mystical-Runic dans votre projet Rust, vous devez l'ajouter comme dépendance dans votre fichier Cargo.toml. Assurez-vous d'avoir la chaîne d'outils Rust (rustup, rustc, cargo) installée sur votre système.

Ajouter la Dépendance

Ouvrez votre fichier Cargo.toml et ajoutez la ligne suivante sous la section [dependencies]. Il est recommandé d'utiliser la dernière version stable disponible.


[dependencies]
mystical-runic = "0.5.2" # Version actuelle

Fonctionnalités Optionnelles v0.5.0+

Mystical-Runic v0.5.2 offre plusieurs fonctionnalités optionnelles pour l'intégration d'écosystème et les fonctionnalités avancées. Activez-les selon vos besoins :


[dependencies]
mystical-runic = {
    version = "0.5.2", # Version actuelle
    features = [
        "async",
        "web-frameworks", # Inclut axum-integration, warp-integration, actix-integration
        "wasm",
        "cli" # Pour les outils CLI si vous construisez une application CLI
    ]
}
  • async : Support pour le rendu asynchrone non-bloquant avec Tokio, essentiel pour les applications web hautes performances.
  • web-frameworks : Intégrations natives pour Axum, Warp, et Actix-web avec réponses HTML directes.
  • wasm : Compatibilité WebAssembly pour le rendu dans les navigateurs avec bindings JavaScript.
  • cli : Outils en ligne de commande pour le traitement de templates, surveillance de fichiers et traitement par lots.
  • full : Active toutes les fonctionnalités d'intégration d'écosystème (async + web-frameworks + wasm + cli).

Fonctionnalités Spécifiques par Framework

  • axum-integration : Intégration spécifique Axum uniquement
  • warp-integration : Intégration spécifique Warp uniquement
  • actix-integration : Intégration spécifique Actix-web uniquement

Exemple d'Utilisation Basique

Une fois la dépendance ajoutée et les fonctionnalités désirées activées, vous pouvez commencer à utiliser Mystical-Runic dans votre code Rust :


use mystical_runic::{TemplateEngine, TemplateContext, TemplateValue};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Initialisez le moteur de template, en spécifiant le répertoire de base pour les templates
    let mut engine = TemplateEngine::new("templates"); 
    
    // Créez un nouveau contexte pour vos données
    let mut context = TemplateContext::new();

    // Ajoutez des données au contexte
    context.set("name", TemplateValue::String("World".to_string()));

    // Définissez votre template en ligne
    let template_content = "Hello {{name}}!";
    
    // Rendez le template avec le contexte fourni
    let rendered_html = engine.render_string(
        template_content,
        &context
    )?;

    // Affichez le résultat rendu
    println!("{{}}", rendered_html);

    Ok(())
}
