Syntaxe de Templating

Mystical-Runic utilise une syntaxe de templating inspirée de Mustache, rendant les templates intuitifs et faciles à lire. Cette section détaille les principaux éléments de syntaxe que vous utiliserez pour construire vos templates dynamiques.

Variables ({{ variable }})

Affiche la valeur d'une variable. Le contenu est automatiquement échappé pour prévenir les attaques de Cross-Site Scripting (XSS), garantissant que le HTML généré est sûr.

{{ nom_de_la_variable }}

Exemple : Si votre contexte contient {"user": {"name": "Alice"}}, alors {{ user.name }} affichera Alice.

Contenu Non Échappé (Raw HTML) ({{& html }})

Permet d'afficher du contenu HTML brut sans aucun échappement. À utiliser avec une extrême prudence et uniquement avec des sources de données fiables, car cela peut introduire des vulnérabilités XSS si le contenu n'est pas sécurisé.

{{& raw_html_content }}

Exemple : Si raw_html_content est <strong>Texte en gras</strong>, alors {{& raw_html_content }} affichera Texte en gras.

Conditionnelles (If/Else) ({{#if}}...{{/if}})

Affiche un bloc de contenu si une condition donnée est vraie. Vous pouvez également utiliser un bloc {{else}} pour un contenu alternatif.

{{#if is_admin}}
    <p>Bienvenue, administrateur !</p>
{{/if}}

{{#if user_logged_in}}
    <p>Vous êtes connecté.</p>
{{else}}
    <p>Veuillez vous connecter.</p>
{{/if}}

Boucles (For) ({{#for}}...{{/for}})

Itère sur une collection (tableau ou objet) et rend un bloc de contenu pour chaque élément. Très utile pour générer des listes ou des tableaux dynamiques.

{{#for item in items}}
    <li>{{ item.name }} (Quantité: {{ item.quantity }})</li>
{{/for}}

Vous pouvez également accéder à l'index de l'itération pour les tableaux :

{{#for item in items with index}}
    <li>{{ index }}: {{ item.name }}</li>
{{/for}}

Inclusions ({{include "chemin/vers/fichier"}})

Permet d'inclure le contenu d'un autre fichier de template dans le template actuel. Utile pour la modularité et la réutilisation de composants.

{{include "partials/header.html"}}
<main>
    <h1>Contenu principal</h1>
</main>
{{include "partials/footer.html"}}

Commentaires ({{! Commentaire }})

Les commentaires sont des notes pour les développeurs et ne sont pas inclus dans le HTML rendu final. Ils sont ignorés par le moteur de template.

{{! Ceci est un commentaire interne au template, invisible dans le HTML final }}

Accès aux Propriétés d'Objet ({{ object.property }})

Accédez aux propriétés imbriquées d'un objet en utilisant la notation par points, similaire à l'accès aux propriétés en JavaScript ou Rust.

{{ user.address.street }}
{{ product.details.color }}

Héritage de Template (Layouts) ({{#extends}}...{{#block}}...{{/block}})

Mystical-Runic supporte l'héritage de template, une fonctionnalité puissante pour créer des layouts de base réutilisables et les étendre dans des templates enfants. Cela favorise la cohérence du design et réduit la duplication de code.

// layout.html (Template Parent)
<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <title>{{ title }} - Mon Site</title>
</head>
<body>
    <header>Mon En-tête</header>
    <main>
        {{#block "content"}}{{/block}}
    </main>
    <footer>Mon Pied de Page</footer>
</body>
</html>

// page.html (Template Enfant)
{{#extends "layout.html"}}
    {{#block "content"}}
        <h1>Bienvenue sur ma page spécifique !</h1>
        <p>Ceci est le contenu unique de cette page.</p>
    {{/block}}
{{/extends}}

Macros Réutilisables ({{#macro}}...{{/macro}})

Définissez des blocs de code réutilisables avec des paramètres, similaires à des fonctions ou des composants. Les macros peuvent être définies une fois et appelées plusieurs fois dans vos templates.

{{#macro "card" title content}}
    <div class="card">
        <h3>{{ title }}</h3>
        <p>{{ content }}</p>
    </div>
{{/macro}}

{{card "Titre de la Carte 1" "Contenu de la première carte."}}
{{card "Titre de la Carte 2" "Contenu de la deuxième carte."}}
